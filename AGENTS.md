# AGENTS.md

## 项目定位

本仓库 (`~/.agents`) 是一个**跨设备 skill 同步中心**。核心目标：在多台电脑上保持 agent skills 的一致性——任何一台机器上安装、更新、删除 skill，都应反映到其他设备。

## 核心规则

### 1. Skill 安装后必须检查 README 目录

**每次安装新 skill 后，必须执行以下流程：**

1. 使用 `find-skills` skill 查找并安装 skill（具体命令见 skill 介绍页面）
2. 读取 `README.md` 中的 skill 目录
3. 对比 `skills/` 目录下实际存在的 skill 文件夹与 README 目录中的条目
4. 如果有差异，更新 README 目录：
   - 新增的 skill → 添加条目（含下载日期、版本、简介、分类、来源）
   - 已删除的 skill → 移除条目
   - 版本变更的 skill → 更新版本号
5. 确保分类准确、信息完整

### 2. Skill 删除后同样必须检查

删除 skill 后，从 README 目录中移除对应条目。

### 3. 定期全量校验

即使没有安装/删除操作，当用户要求时，执行全量校验：
- 扫描 `skills/` 下所有子目录
- 逐一与 README 目录比对
- 输出差异报告并更新

## Skill 信息采集规范

从每个 skill 的 `SKILL.md` frontmatter 中提取：

| 字段 | 来源 | 说明 |
|------|------|------|
| 名称 | `name` 字段 | skill 的唯一标识 |
| 版本 | `version` 字段 | 无 version 则标注 "latest" |
| 简介 | `description` 字段 | 精简为一句中文描述 |
| 下载日期 | 安装当天日期 | 格式 YYYY-MM-DD |
| 来源 | 安装命令中的 `owner/repo@skill` | 记录上游来源 |
| 依赖 | `metadata.requires` | 如 `lark-cli`、`curl` 等 |

## 分类体系

Skill 按功能分为以下类别。**分类体系可扩展**：当新增 skill 无法归入现有分类时，应新增分类并更新本表。

### 新增分类规则

当出现以下情况时，应新增分类：
1. 新 skill 的功能领域与现有所有分类都不匹配
2. 某领域 skill 数量超过 5 个，且有独立业务价值

新增分类步骤：
1. 在上表中添加新分类（含 emoji 图标和说明）
2. 同步更新 README.md 的分类说明表
3. 将相关 skill 归入新分类

### 来源标识

README 目录中的 skill 条目应标注来源信息：

| 来源类型 | 标识 | 示例 |
|----------|------|------|
| 大厂官方 | `🏢 官方` | Anthropic、OpenAI、字节跳动、阿里等 |
| 大厂非官方 | `🏢 非官方` | 大厂员工个人开发 |
| 社区开源 | `🌐 社区` | 开源社区贡献 |
| 个人开发 | `👤 个人` | 个人开发者 |
| 未知 | `-` | 无法确定来源 |

**大厂定义**：知名科技公司（如 Anthropic、OpenAI、Google、Meta、字节跳动、阿里巴巴、腾讯、华为等）

## 注意事项

- README 目录中的 skill 排序：先按分类，同分类内按名称字母序
- 下载日期以首次安装日期为准，更新 skill 时不修改原始下载日期，但记录更新日期
- 如果 skill 的 SKILL.md 没有 version 字段，版本栏标注 `-`
- 来源信息如无法从安装记录获取，标注 `unknown`
