# Image Output Registration Contract

Every target-paper image must be generated by the required environment image route and registered separately. Do not replace separate sketches or candidates with a contact sheet, stitched board, SVG/HTML/canvas/Mermaid/PPT-shape substitute, or hand drawing.

Canonical active image output patterns are resolved from the candidate registry and stage manifest. The default scaffold paths are:

- S2-SKETCH-EXPLORE: `outputs/S2-sketch-explore/candidates/C01/image-v01.png`, `outputs/S2-sketch-explore/candidates/C02/image-v01.png`, and so on, unless state/manifest registers a different active image path.
- S5-CANDIDATE-IMAGE: `outputs/S5-candidate-images/candidates/C01/image-v01.png`, `outputs/S5-candidate-images/candidates/C02/image-v01.png`, and so on, unless state/manifest registers a different active image path.
- S6-FINAL-SELECT: records one selected S5 raster candidate as `s6.selected_reference_final` and registers `outputs/S6-final-selection/final-figure-contract.md` as `s6.final_figure_contract`.
- S7-FINAL-JOINT-AUDIT: copies or registers the selected reference as `outputs/S7-final-joint-audit/pending-submission-figure.png` (`s7.pending_submission_figure`) before audit; if final-figure PASS, copies/registers it as `outputs/S7-final-joint-audit/submission-final-figure.png` (`s7.submission_final_figure`), then generates/registers `outputs/S7-final-joint-audit/element-icon-inventory.md` (`s7.element_icon_inventory`), one or more `outputs/S7-final-joint-audit/submission-element-icon-sheet-XX.png` sheets (`s7.element_icon_sheet_primary` for sheet 01), and `outputs/S7-final-joint-audit/icon-sheet-audit.md` (`s7.icon_sheet_audit`).

S2 default and minimum count is 8 sketches; maximum is 8.

S5 default count is 6 formal candidates; maximum is 8.

S5 candidates remain raster images. They should use clean modules, paper-relevant icons, short labels, precise connectors, meaningful color, and a style-aware caption plan. Later SVG/PPT approximation is secondary and must not drive icon choice.

For `runtime=chatgpt_web`, S2/S5 use one full image batch when the platform supports it: default S2 uses `C01-C08`, and default S5 uses `C01-C06`. Split only when the platform, user, or current failure mode requires it. S7 image actions are registered one image at a time. The next user prompt for each image-only unit must be saved by a text unit under `substage-guides/`; image-only units do not write audit or next-step prose.

Do not put `Expected active image paths` or similar path lists inside user-facing copyable prompts. The skill resolves relative output paths from `stage-manifest.json`, `candidate_run_registry`, `image_generation_events`, and checkpoint manifests. Exact path lists belong in machine-readable state/manifest files, and in `checkpoint-missing-images.json` only when a checkpoint is incomplete.

After an S7 image-level failure, repair stays inside S7: archive the failed pending-submission figure and failed audit/spec, generate exactly one repaired raster replacement at `outputs/S7-final-joint-audit/pending-submission-figure.png`, register it as `s7.pending_submission_figure`, overwrite canonical S7 audit/spec with the latest audit, and require a final S7 `PASS` before creating a submitted final figure.

The S7 element icon sheets are also target-paper raster deliverables. They must be generated by an approved image route from the locked final figure's element inventory, not coded as SVG/HTML/canvas and not edited from failed sheet rasters. They must contain isolated reusable icons/glyphs with enough blank margin for cutting, and no connector lines, arrows, callout lines, text, formulas, mathematical symbols, or full-figure layout.

Example generation-event command:

```bash
python scripts/figure_studio_state.py record-image-generation \
  --project-id <project_id> \
  --event-id s5-candidate-001 \
  --step S5-CANDIDATE-IMAGE \
  --batch-id s5-candidates \
  --generated-path outputs/S5-candidate-images/candidates/C01/image-v01.png
```
