# Changelog

## v3.1.6a

- Hardened ChatGPT web text-only handoffs: every user-visible text-only prompt now ends with the exact `本轮纯文字` guard, and saved guidance auto-appends the guard for text substages.
- Removed user-maintained relative path lists from copyable prompts: expected active image paths are stored in state/manifests or missing-image checkpoint manifests, not pasted by the user.
- Hardened cumulative checkpoints: stage-final checkpoints are explicitly cumulative through the current stage, required S2/S5 candidate images are checked even when only default paths are available, missing images mark the checkpoint incomplete, and optional numbered zip parts are supported through `--max-zip-bytes`.
- Clarified S2/S3/S4 responsibility split: S2 audit is a semantic risk ledger and prompt-risk discovery source, not S3's positive ranking authority. S3 now independently selects paper-grounded directions from S0/S1 logic and S2 visual exploration signals, while transferring S2-audited error patterns into S4/S5 prompt constraints.
- Generalized S2/S5 lightweight and strict audit rules so they are driven by S0 paper evidence, risk-register constraints, and candidate contracts rather than hard-coded federated, semi-supervised, raw-data-sharing, or test-set assumptions from one example paper.
- Reworded connector, topology, area-budget, multi-space balance, core-module opacity, and lineage audit policies to use paper-defined role/topology/artifact/evaluation constraints while preserving the existing `final_only` default, optional strict S2/S5 checks, audit-only default behavior, and user-preauthorized one-repair limit.
- Updated package metadata, README, OpenAI agent metadata, release listings, and project-state template version to `3.1.6a`.

The first S7 unit must run the complete final audit, save the next guidance, update internal-run state, and stop; completion is legal only after later lock/spec, icon inventory, icon-sheet image, icon-sheet audit, and final aggregate units have completed.
- Normalized workflow naming without changing the version: S2/S5 metadata modes now match `DYNAMIC_TEXT_IMAGE_SUBSTAGES`, the S7 policy file is `s7-internal-workflow-policy-v316.md`, obsolete S7 internal modes were removed, and `S7-*` labels are explicitly verdict labels rather than substage or internal-run mode names.

- Tightened S7 multi-repair lineage without changing the version: each S7 final-figure repair attempt must use the latest failed canonical `pending-submission-figure.png`, the latest failed S7 audit/spec, and the source prompt/brief that produced that failed pending image. After a repaired pending image exists, later repairs must not fall back to the original S6 selected raster.

- Added arrow-direction semantics: arrows point to information destinations or next steps; reverse, unsupported, decorative, redundant, or ambiguous callout arrows are semantic failures.
- Made S7 final contract fidelity explicit in the S7 report template: the bounded audit table now has a dedicated `Final contract fidelity` pass matching the mandatory S6 `final-figure-contract.md` gate.
