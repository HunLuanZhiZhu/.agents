# Zuhui PPT Swiss Design（中文说明）

## 目标

把密集论文内容做成高级感瑞士风组会 PPT，视觉规则已内化，无需继承外部 skill。

## 必须继承的 Guizang 规则

- HTML 优先使用本 skill 目录下的 `assets/template-swiss.html`（位于 `zuhui-ppt-master/assets/`）；
- 使用瑞士国际主义：Inter/Helvetica/Noto Sans SC、网格、大字号、IKB 蓝、柠檬/荧光强调；
- 遵守 Swiss locked mode；
- 正文页必须来自 `S01–S22` 注册版式或允许的封面/尾页扩展；
- 每个 `<section class="slide">` 必须写 `data-layout="Sxx"`；
- SVG 逻辑图：矩形、箭头、线条，无阴影无圆角，使用 Swiss 配色，每份 deck 1-3 张。
- 普通顶部标题不要居中；
- 不要发明未登记正文版式。

## 可见语言规则

- PPT 可见文字默认中文。
- 英文只能作为缩写、代码/仓库标识，或放在中文解释后的括号里：`受控评测（controlled evaluation）`。
- 不要使用裸露英文 kicker、章节标签、caption、页脚或卡片标签，例如 `WHY NOW`、`CLAIM`、`WORKFLOW`、`Q&A`、`controlled evaluation`。
- 截图中的源图标签可以保留，但页面旁边必须就地翻译难词。

## 默认主题

默认使用 IKB 克莱因蓝。

推荐节奏：

- IKB 蓝封面；
- 暗色问题 statement；
- 白底图表证据页；
- 每个大章节用暗色幕封；
- 结尾用暗色或 IKB；
- 版式混用 `S09`、`S18`、`S21`、`S22`、`S19`、`S08`、`S20`。

## v1 视觉金标准

如果用户明确更喜欢早期 v1，就把 v1 当作视觉金标准，不要继续沿用被批评的 v3/v4/v5 风格。本工作流中 v1 风格指：

- Guizang `template-swiss.html` 加薄的项目 CSS 层，例如 `aris-*`；
- IKB 封面和尾页、黑色 statement/章节页、白底证据页；
- 细边框卡片、大留白、高对比瑞士字体；
- 大图 hero 页中图片占主导，文字只是窄解释栏；
- 不使用包含大段论文正文或图注的装饰性截图。
- 不让英文单独出现在可见设计框架中，除非已经配中文。

如果 v3/v4/v5 风格和 v1 冲突，下一版必须回到 v1 视觉语言；增加信息量应靠更密集的证据和表格，而不是更重的装饰。

## 版式映射

- 封面：`SWISS-COVER-ASCII` 或强 Guizang Swiss 封面；
- 问题页：`S09` 或 `S03`；
- 密集表格：`S21`；
- 大图/大表：`S22`；
- 架构/工作流：`S22` 或 `S17`；
- 三概念：`S13`；
- 四特性/局限：`S19`；
- 对比：`S08`；
- 指标页：`S20`；
- 章节页：暗色 `S09`；
- 收尾：`S10` 或 `SWISS-CLOSING-ASCII`。

## 图片大小规则

论文证据页中，图或表通常应该占 55–80% 页面面积。

做法：

- 用窄解释栏，而不是 50/50 平分；
- 工作流、对比表、附录图优先全宽或近全宽；
- 图太密就裁剪或拆页，不要缩小；
- caption 贴着图；
- 底部保留来源。

## 截图和表格规则

- 截图必须是纯图裁剪：不能带正文段落、图注、表注或 PDF 周围文字；
- 有论文 `images/` 文件夹时，优先用其中干净图片作为裁剪边界参考；
- 表格默认重建为原生 HTML/PPTX 表格，除非它本身和图形不可分离；
- 翻译图只裁图本体，长 caption 改写成可编辑中文解释；
- 干净的原生表格优先于低清晰度表格截图，即使因此多拆几页。

如果用户说图太小，下一版必须显著增大图片占比，否则算失败。

## 图信息量评估规则

不是每张图都值得独占一页。使用 S22 Image Hero 或给图单独一页前，先评估：

### 适合全页/全宽的条件

1. **图类型**：架构图、工作流图、多面板对比图、复杂系统图通常信息量够；简单柱状图、单线图、单面板示意图通常不够。
2. **内部信息含量**：含 3+ 子图、多个数据维度、复杂标注的可以全页；单一数据维度、简单趋势线的不行。
3. **论文角色**：核心证据图（main result、关键 ablation、中心机制）可以全页；辅助示意图、背景介绍图应与文字或其他图组合。

### 低信息量图的处理

- 与文字和讲稿组合为 text-led synthesis 页（S13、S19）
- 与相关图并排对比（S08、S21）
- 缩小为多卡片布局中的卡片（S16、S19）
- 作为文字页中的小插图

### S22 使用条件

S22 Image Hero 仅当图通过信息量评估后才使用。信息量丰富且是论文核心论据的图才适合 S22。

## SVG 逻辑图绘制指南

对非照片类图表，手绘内联 SVG 优于论文截图。每份 deck 控制在 1-3 张。

### 适用场景

- 架构图、工作流图、模块关系图、对比框架、方法论流程图
- 能用矩形、箭头、线条表达的简单几何逻辑图
- 论文原图分辨率低、过于杂乱或有英文标注待翻译

### 不适用场景

- 数据图表（柱状图、折线图等）→ 用论文截图
- 照片、显微图、渲染图 → 用论文截图
- 太复杂的图 → 用论文截图

### 设计原则

- **极简几何**：只用矩形、箭头、线条。无阴影、无渐变、无圆角
- **Swiss 配色**：强调色用 accent，边框和背景用灰度
- **字体**：SVG 内文字用 Noto Sans SC，不小于 14px
- **内联嵌入**：SVG 直接嵌入 HTML section，不外部引用
- **中文标注**：所有可见文字为中文

## 页面充实规则

幻灯片应看起来完整而非空洞。大多数页面应有：

- **顶部锚点**：kicker 或章节标签
- **主导中间块**：证据图、表或 claim 卡片
- **底部锚点**：takeaway 条、来源条、结论行

### 内容密度标准

- 每页至少 2-4 条具体信息点（数据、论据、对比结果）
- 三栏卡片页每栏至少 2-3 句实质性内容，禁止"一句话卡片"
- text-led 页至少 3-4 条带数据或证据的 bullet
- 深层解释移到讲稿是**例外**而非默认，优先在页面上保留小结行

### 信息量自检

- 这页是否有足够的具体数字、对比或论据支撑？
- 不看讲稿能理解页面要点吗？
- 有没有卡片或区域只有标题没有内容？

## 两遍字号法

### 第一遍：全局统一

按字重阶梯、中文标题字号分档和最小可读尺寸统一设置。

### 第二遍：逐页微调

视觉评审后，根据每页内容密度单独调整：

- 内容密集页：适当缩小字号（但不低于 18px/16px/14px 底线）
- 内容稀疏页：适当放大字号填充空间
- 调整后仍需遵守字重阶梯和标题分档规则

## KaTeX 公式渲染

HTML 中的数学公式使用 KaTeX 渲染。

### 引入方式

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.css">
<script src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/contrib/auto-render.min.js"></script>
```

### 写法

- 行内公式：`\(E = mc^2\)`
- 独立公式：`$$\mathcal{L} = -\frac{1}{4}F_{\mu\nu}F^{\mu\nu}$$`
- 公式颜色：`\color{#002FA7}` 匹配 IKB 强调色
- 独立公式用 `\Large` 或 `\LARGE` 确保投影可读

### 降级

KaTeX CDN 不可用时用 MathJax 3.x。

## HTML 构造

优先流程：

1. 读取 `zuhui-ppt-master/assets/template-swiss.html`；
2. 替换标题占位符；
3. 注入最少项目 CSS；
4. 把 slide `<section>` 插入模板区域；
5. 每页保留 `data-layout` 和 `data-animate`；
6. 保留 Guizang 翻页、导航、WebGL/网格背景和 overview。

## PPTX 构造

PPTX 不需要复刻动画，但要保留：

- IKB / 暗色 / 白底节奏；
- 大图比例；
- 标题、正文、caption 层级；
- 可编辑文字；
- 必要时使用原生表格；
- 来源 footer 和页码。

## 设计 QA

导出前检查：

- 主题节奏；
- 是否连续多页空泛；
- 25 页以上至少 8 个 Swiss layout ID；
- 图是否足够大；
- 图表页是否有 caption 和来源。
