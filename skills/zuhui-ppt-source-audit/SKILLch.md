# Zuhui PPT Source Audit（中文说明）

## 目标

在设计任何页面之前，先把论文真实证据抽出来，避免 PPT 变成空泛三栏页。

## 要检查的输入

- 原始论文 PDF；
- 翻译 PDF，尤其是里面已经翻译过的图和表；
- 旧版本 PPTX / HTML / PDF / 脚本 / QA；
- `paper-ppt-agent` 的输出和工作区；
- 用户反馈和硬约束；
- 用户指定的个人信息。

## 翻译 PDF 规则

只要有翻译 PDF，就要假设其中可能已经包含翻译后的图表。即使没有独立图片文件，也可以：

1. 渲染低分辨率页面 contact sheet；
2. 找到关键图表页；
3. 高分辨率渲染选中页面；
4. 只裁出中文图或流程图本体到 `assets/`，不要把正文段落、图注、表注一起裁进去；
5. 生成裁图 contact sheet；
6. 记录页码、文件名、图表身份和计划用途。

不要把“如何解决英文图”写进 PPT。正确做法是直接使用干净图裁剪并在页面旁边解释。表格应该根据论文文字重写成原生表格，不要截图。

如果有论文 `images/` 目录，要把它作为裁剪范围参考。优先使用这些干净图，而不是包含图注/正文的大范围 PDF 截图。

## 旧版本审计

检查：

- 页数；
- 是否有 Guizang Swiss 风格；
- 图片是否太小；
- 页面是否空；
- 是否有用户讨厌的元内容；
- 是否缺少论文表格、附录和实现细节；
- 是否保留 PPTX / HTML / PDF。

用户批评就是验收标准。

## 学习 paper-ppt-agent

如果有参考项目，要看：

- `manuscript.md`：密集中文内容和主线；
- `design_spec.md`：页面密度、边距、字体、证据对象；
- 输出 PPTX / HTML / SVG：视觉节奏；
- pipeline：strategist → executor → critic → export。

学习流程，不要抄没有论文支持的数字。

## 输出

生成源文件审计记录：

- 源文件路径；
- 论文元数据；
- 论文类型；
- 翻译 PDF 页码地图；
- 候选图表；
- 已选资产；
- 每个视觉对象是干净图、原生表格，还是被拒绝的大范围截图；
- 旧版本问题；
- 用户限制和禁用内容；
- 缺失或风险。

## 质量门槛

必须完成：

- 有用的翻译图表已经裁出；
- 包含正文或图注的大范围截图已经剔除；
- 需要展示的表格已经安排原生重写；
- 裁图 contact sheet 已检查；
- 旧版本问题明确；
- 附录、表格、实现细节进入候选证据地图。
